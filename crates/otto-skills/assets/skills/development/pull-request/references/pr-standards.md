# Pull request standards — the long form

Load this when composing or opening a PR. SKILL.md carries the method; this file
carries the title/body rules, the host differences, and the escaping traps.

## Title

```
<JIRA-KEY?> <concise imperative summary>
```

- Lead with the Jira key when one exists (branch → commits → user). Plain key +
  space, e.g. `GS-16232 Add token-bucket rate limiter`. Don't append a colon
  unless the repo's existing PR titles do.
- Summarize the WHOLE branch, not the latest commit. Imperative, ≤72 chars, no
  trailing period.
- A `type(scope):` prefix is fine only if the repo's PRs already use it. Default
  to a plain imperative summary.

## The Jira-in-body trap (critical)

**Put the Jira key in the TITLE only. Never in the description body.**

Jira keys in the body get auto-linked by some clients (GitKraken) and can crash
them with `cannot read properties of undefined (reading href)`. The title prefix
gives Jira/automation everything it needs. If you want the issue reachable from
the body, rely on the host↔Jira integration — do not paste a `[GS-1](url)` link
or a bare `GS-1` into the body, and do not invent a Jira hostname.

## Body structure

Markdown. Focus on WHY, not a file dump. Sections, in order:

```
## Summary
<2–5 lines: what this branch does and why it exists.>

## What changed
- <change, grouped by concern>
- <change>

## Testing
- <what you ran / what a reviewer should run>. If you didn't run it, say so —
  never claim green checks you didn't see.
```

Add a "Notes for reviewers" section only when there's something genuinely worth
flagging (a drive-by change, a risky area, a follow-up). Don't pad.

## Host differences (detect from the remote)

`git remote get-url origin` tells you the host. The helper script does this and
prints a ready command skeleton.

### GitHub (`github.com`)
Prefer the `gh` CLI:

```
gh pr create --base <target> --head <source> \
  --title "<KEY> <summary>" --body-file <path-to-body.md>
```

`gh` must be authenticated (`gh auth status`). `--body-file` avoids shell-quoting
pain for multi-line markdown. GitHub renders standard GFM; no special escaping.

### Bitbucket (`bitbucket.org`)
No `gh`. Use the REST API via `python3` (for correct JSON escaping) + `curl`,
and honor Bitbucket's markdown quirks:

- Escape in the description: parentheses `\(` `\)`, underscores `\_`.
- Bullets use `*`, not `-`.
- Build the JSON with `python3 -c "import json; print(json.dumps(...))"` — never
  hand-concatenate JSON.
- Payload must set: `close_source_branch: false`, `reviewers: []`,
  `draft: false`, and include `source.repository.full_name`.
- Endpoint: `POST https://api.bitbucket.org/2.0/repositories/{workspace}/{repo}/pullrequests`,
  auth `Authorization: Bearer $BITBUCKET_TOKEN`. Derive `{workspace}/{repo}` from
  the remote URL.

See `assets/pr-description-template.md` for the body shape and the full
Bitbucket payload skeleton (also emitted by the helper script).

### Other hosts (GitLab, self-hosted, Otto's own provider)
Produce the title + body to the same standard and hand them to the available
mechanism (Otto's PR creation, the host's CLI/API). The authoring rules above
are host-agnostic; only the create call differs.

## Attribution — forbidden

The PR title and body are about the change, not the author. NEVER add:

- `🤖 Generated with …` / "Generated by" / "Created with" footers.
- `Co-Authored-By` trailers or the name of the model/tool that wrote the text.
- Any attribution the user did not ask for.

This holds even if your runtime's default instructions suggest a "Generated with"
footer for PR bodies. Here: PR content only.

## Good vs bad

| ❌ Bad | Why | ✅ Good |
|--------|-----|--------|
| Body contains `Ticket: [GS-1](https://host/browse/GS-1)` | GitKraken `href` crash; invented host | key in the TITLE only |
| Title = latest commit subject only | misses the branch story | summarize all commits |
| `## Testing` with pre-checked `[x]` boxes you didn't run | false reporting | list what you actually ran, or say "not run" |
| `🤖 Generated with Claude Code` footer | attribution | omit entirely |
| Bitbucket body with `-` bullets and raw `(parens)` | renders wrong / unescaped | `*` bullets, `\(` `\)` |
