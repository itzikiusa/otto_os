## D. FolderPicker — search + hidden toggle
- `FolderPicker.svelte`: add `let filter = $state(''); let showHidden = $state(false);`
- A filter `<input>` above the list (placeholder "Filter…", autofocus). A small "Show
  hidden" checkbox/toggle in the header row.
- `const shown = $derived((view?.entries ?? []).filter(e => (showHidden || !e.name.startsWith('.')) && (filter.trim()==='' || e.name.toLowerCase().includes(filter.trim().toLowerCase()))));`

## E. Graph render fix (the "wall" / break)
Root cause: in the lane allocator, when MORE THAN ONE lane awaits the same parent sha
(shared history / repeated merges in an `--all` log), `lanes.indexOf(sha)` resolves only
the FIRST; the other lanes keep waiting forever and pile up into a wall of dangling
verticals lower in the graph. Fixes in the `laneRows` `$derived.by`:
1. **Collapse duplicate-awaiting lanes:** after resolving `col` for `commit.sha`, scan
   `lanes[]` for any OTHER index `i` with `lanes[i] === commit.sha`; set `lanes[i] = null`
   (freed) AND push a converge line `{fromCol:i, toCol:col, color: laneColor(i), kind:'merge-in'}`
   so the graph SHOWS them converging instead of running parallel forever.
2. **Trim trailing free lanes:** after updating lanes for the row, `while (lanes.length && lanes[lanes.length-1] === null) lanes.pop()` so the gutter width shrinks back when branches close (keeps `gutterWidth` bounded).
3. **Safety cap (defensive):** if `lanes.length` exceeds a max (e.g. 24), stop allocating
   NEW lanes for extra/merge parents and route them to the node's own column (the graph
   degrades gracefully instead of exploding). Log nothing; just cap.
4. Keep node color = its lane color; keep the existing vert/merge-in drawing. Verify
   `gutterWidth` is derived from the *max* lane count seen (it already caps at ~200px).
Add/extend an in-component derivation; no API change. (Optionally raise the log request
limit from 200 toward 400 so more merge-bases are reached — but the collapse fix is the
real one; keep 200 to avoid perf regressions unless trivial.)
